gate.cat

Stop your AI agent before it does the irreversible thing.

A deny-list that sits between an agent and the real world — rm -rf, terraform destroy, force-pushes, payments, outbound email. Deterministic rules, fail-closed, every decision logged.

$ pip install gate.cat

A veto in three lines

from cacheback.integrations import check_action, ActionVetoed
from cacheback.integrations.policies import DOGFOOD_DEFAULTS

check_action("agent", "terraform destroy -auto-approve", DOGFOOD_DEFAULTS)
# -> ActionVetoed: VETO [TERRAFORM_PROD] — the action never ran

Drop-in for Claude Code (a PreToolUse hook), crewAI and LangGraph. The agent proposes; the gate decides — allow, or block and escalate to a human.

Why deterministic, not an LLM judge

It's rules, on purpose

A firewall is "just rules" and that's exactly why you trust it in the enforcement path. A probabilistic guard can hallucinate a pass on the one terraform destroy that mattered.

Fail-closed

Engine error, weird input, unreadable decision → block, never a silent allow. An unverifiable action is never waved through.

Audited

Every decision — allow and block — is one JSON line in ~/.cacheback/veto_log.jsonl. Prove after the fact what the gate saw.

Built for small/local models

Aimed at the cheap/local agent stack (7–30B via Ollama/vLLM) that frontier-first guardrail vendors don't target.

Honest by design

The gate is certain only about what it blocks. An action it doesn't match is unchecked, not safe. The published bypass suite maps exactly what it catches and what it misses — no "detects lies", no "guarantees safety".
